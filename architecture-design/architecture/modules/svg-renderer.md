# SVG Renderer

## 模块定位

SVG Renderer 负责绘制架构图主体，包括对象节点、组合模块外框、关系折线、箭头、hover 关系说明气泡和文档跳转链接。

## 当前实现

当前实现位于 `scripts/render_arch_graph.py`：

- `render_object()` 绘制节点，并在对象存在 `ref` 时生成节点文档链接。
- `render_group()` 绘制组合模块外框和角标，并在组合存在 `ref` 时生成组合文档链接。
- `render_relation()` / route helpers 绘制有向关系折线，并把关系说明渲染为 hover 气泡。
- `render_svg()` 汇总 SVG，并把样式内嵌到 SVG 中，保证独立 `.svg` 文件也能直接查看。

## 当前规则映射

渲染器会根据被引用文档的 `module_kind` 自动选择默认图形和颜色；JSON `objects[]` 条目仍可手动覆盖 `shape`、`stroke`、`fill`。

大型真实项目图默认使用更宽松的节点、网格和 SVG 自然宽度。HTML 页面允许横向滚动，避免把大图压缩到窄容器导致文字挤在一起。CLI 可通过 `--svg-output` 同步输出独立 SVG。

节点和 hover 气泡文本按显示宽度换行，中文字符按宽字符处理，避免描述文字超出节点或气泡边界。关系线默认不显示文字，关系说明只在鼠标移到对应关系线附近时显示。

组合模块框按行内连续节点生成分片边界，而不是一律使用覆盖全部子节点的大矩形。这样跨多行或包含空洞的组合不会压到其他组合区域；相邻组合分片之间保留最小间隔，标题只放在第一个分片内。

## 关系路由与 hover 说明

关系线使用混合正交折线路由，不再使用斜线。路由先按关系位置生成快速候选路径，再为复杂关系追加正交可见轨道搜索候选，最后统一打分选出最低成本路径：

- 同一行关系优先水平直连；跨多个节点时可走上方或下方轨道。
- 同一列关系优先垂直直连；跨多行时可走左侧或右侧轨道。
- 斜向关系生成水平优先、垂直优先、L 型和 Z 型候选。
- 长距离或回流关系可走图外侧走廊，减少穿过中心区域。
- 搜索候选把节点边界、组合标题区、端口和图外走廊转换为 Manhattan 轨道，用 Dijkstra/A* 风格的低成本路径搜索避开障碍。
- 候选路径会按是否穿过节点、组合标题区、是否与已有线段长距离重叠、折点数量和路径长度评分；短端口共用段只做轻量惩罚。
- 关系端口落在模块图形的上、下、左、右边界点，并在边界外补一小段法向连接桩，保证起点贴住模块边界、箭头最后一段朝向目标模块内部。
- 关系线和箭头使用近黑色，避免与蓝色组合底框或节点类型色混淆。

每条关系会输出两类 SVG path：

- 可见 path：细线和箭头，位于节点层下方。
- hover hitbox path：透明粗线，位于节点层上方，负责鼠标命中。

hover 时显示消息气泡，内容来自 `relations[].described`。气泡自动避开节点框和组合标题区；JSON 中的 `label_offset` 仍可作为气泡锚点微调。

## 跳转行为

对象节点和组合模块外框的点击跳转由 SVG Renderer 负责。它不重新定义图结构，只消费 Graph Model 中已经解析好的 `ref` 字段，并按输出 HTML 的位置优先转换为相对文档链接。

- `DiagramObject.ref` 存在时，整个节点可点击并打开对应 Markdown 模块文档。
- `DiagramGroup.ref` 存在时，组合框可点击并打开对应组合模块文档。
- 无 `ref` 的节点或组合只渲染为普通图形，不生成链接。
